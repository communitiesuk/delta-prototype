UX design and spec for data range selector for us in filters throughout Delta

https://mhclgdigital.atlassian.net/browse/DT-1292

Post Brian chat, 24 October 2024…

  • Dates to be entered as numbers in text fields as per GDS guidance (new window)
  • Examples:
    • 1 1 2024 (would be 1 January 2024)
    • 01 1 2024 (would be 1 January 2024)
  • Example dates should be displayed
  • Never user a date picker / calendar (loads of accessibility issues)
  • Beauty of this system is doesn't require javascript
  • Text on apply filters should be dynamic. If no filters active it should say "Apply filters". If some filters active it should say "Update filters"
  • "Clear filters" could be made less prominent (i.e. a text link) AND should only be displayed if some filters are active
  • Error checking
    • You must enter a from date
    • You must enter a to date or select all future dates
    • Confirm to date is after from date, and vice versa
  • Add link to 'clear dates' - only display if date range has been entered

Design notes

  • 'To date' and associated hint text and inputs only displayed if user selects radio button 'specify date'
  • Q: would it be better to use a checkbox for 'All future dates' and display the future date input fields if this is unchecked?
  • 'Clear all dates' link only displayed if any date range information has been entered
  • 'Clear filters' link only displayed if there are filters to display
  • 'Apply filters' button should say 'Update filters' if changes to filters have been made

Data range picker requirements

  • User can specify day*, month, year for From and To
  • Dates follow format: Day (number) Month (word) Year (number).
    • The day is 1-31 with 0 omitted for days less than 10.
    • Months are written out.
    • Years are always 4 digits.
    • Example: 1 January 2923 or 31 October 2024.
  • For Delta where possible we want to allow users to avoid having to type in data. Instead we want to offer drop-down controls.
  • We also want to offer calendar date picker
  • There should be option to clear all data
  • Needs to be accessible
  • Needs error reporting:
    • You must enter from date
    • You must enter to date
    • Confirm to date is after from date, and vice versa
  • Accessibility
    • Highlight fields that have missed data
  • Drop-down considerations?
