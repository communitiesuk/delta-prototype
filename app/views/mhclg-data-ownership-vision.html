MHCLG data ownership

A proposal for a new way of working with data assets in MHCLG.

Updated: 19 September 2024

What is a data asset?

  • Any data collected by MHCLG. Examples: A, B and C
  • Any data built upon data from MHCLG or others. Examples: X, Y and Z

Why was this work undertaken?

To transform the way MHCLG works with data

  • Provide a streamlined and efficient approach to managing data.
  • Establish a clear understanding of existing data assets, their maintenance, and how they are utilised.
  • Enable fast, innovative methods for delivering insights to the Deputy Prime Minister and the broader public sector.
  • Ensure compliance with legal obligations related to data standards.

The Data Catalog

A comprehensive inventory of all data assets managed by MHCLG.

For each asset, the following information will be recorded:

  • Detailed description of the data, including available data schemas.
  • Information on data provenance, such as sources and methods of derivation.
  • Usage rights and any relevant commercial considerations.
  • Links to additional resources and the location where the data is stored.
  • Dates of the most recent update and the next scheduled review.
  • Contact information for further enquiries.

The Data Catalogue serves as a central directory, providing robust search and reporting capabilities. It does not host the data assets themselves; these will continue to reside in their respective systems.

Success

The Data Catalogue only works if all data assets are registered and entries actively maintained.

To achieve this we need…

  • New, clear, simple roles with regards to data ownership.
  • Effective monitoring tools.
  • Working practices that reduce, not add to, the burden for data owners.

Data Guardian

Responsible for a data asset​

The Data Guardian is responsible for the day-to-day operational activities needed to support the data governance decisions made by SROs in their data domain.​

  • Domain knowledge: Moderate to high​
  • Data literacy: Moderate​
  • Role level: SEO and above
  • Equivalent roles: CDDO - Data Steward, IAR - Information Subject Matter Expert

Primary roles​

  • Facilitating data management
  • Facilitating data quality
  • Facilitating data compliance

Facilitating data management​

  • Acquire, store, and enable usage of data in line with MHCLG policies including security and cyber​
  • Be the operational point of contact
  • Relationship Management: understand and manage data lineage/flow to understand and mitigate impacts of changes, including for statistical products based on the data​
  • Consult stakeholders regarding the impacts of any changes to the data, e.g. ensure that statistical products can still be generated based on the data​
  • Handle data governance queries (checking with SRO for tactical guidance if needed)​
  • Draw up data share agreements including seeking legal advice and work with other Data Guardians to resolve cross cutting issues/risks​
  • Define the data access controls​
  • Identify the secure tools needed for data collection and provision the acquire, store, and manage the data​
  • Update the Information Assets Register (IAR) and MHCLG Data Catalogue records​

Facilitating data quality

  • Responsible for data interoperability through application of existing data standards and reuse of reference data
  • Responsible for resolving data quality issues
  • Periodically report on value attained from the collected data
  • Ensure that the data and reference data is kept up-to-date

Facilitating data compliance

  • Facilitate governance processes for data access, data archival and data deletion ensuring appropriate retention periods are in place for all data
  • Facilitate data quality governance processes for monitoring, investigating, communicating, triaging, remediating & reporting
  • Report to the SRO and other forums on compliance, issues, fixes and changes
  • Create processes, procedures and standards in the data domain (defining privacy statements)
