Detla is a data collection and grants administration tool. The platform comprises of two channels:

  • Collections â€” used by a mix of internal and external users (Data Providers and Data Certifiers).
  • Grants - used by internal users only.

Collections overview

  • Collections, created by DLUHC teams (internal users), are requests for data.
  • Data is submitted in the form of records by external users.
  • Collections have a data period which are the from and to dates for which data is required. Example: for the question "How many hedgehogs did you see in 2023?" the data period would be 1 January 2023 to 31 December 2023.
  • Collections have a data window. This is the period of time the collection is open for records to be submitted.
  • Collections have two types - record-based (multiple records - e.g. traveler's survey) or summary-based (one record - e.g. annual return)
  • Records can be submitted by 3 methods: form, bulk upload (from Excel template) or API (XML). The options are set by the Collections Manager on a collection by collection basis.

Collections user tasks (aka 'jobs to be done')

External users

Data provider

  • Find the collection I need to provide data for
  • Understand what data is required for which period
  • Understand how long I have to submit data
  • Submit data
  • See responses I have previously made (check data)
  • Understand who else can / has submitted data for a collection
  • See which collections I have access to
  • See if there are other collections I should have access to / join
  • Change my role (provider and certifier)
  • Manage my account (including change my organisation)

Data certifier

  • See which data (records) I need to certify
  • Certify a record
  • Contact the record owner (submitter) if data is incorrect and/or I have questions
