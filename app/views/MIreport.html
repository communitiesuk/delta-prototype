What it is and what it isn't

What problems does this solve?

  • Accessible to all*: For the first time ever one-click access to a global list of all collections with powerful filtering and real-time insights.
  • Public-facing collections: One-click access to view the public-facing collection entity page.
  • Form complexity: A high-level overview of the form's complexity, based on the amount of data requested.
  • Collection channel and data upload options: Ability to view and filter by collection channel (for example, Funding Service) and supported data upload options.
  • Submission data: High-level submission data with links to the submission report.
  • Record store view: One-click access to a filtered record store view for the collection.
  • Errors: Error rates provide insights to support collection improvements and enable timely, targeted support from the relevant collection manager.

Collection MI report

Help with table labels
  • Collection name: The last date any organisation can submit a return. This includes extensions.
  • Status:
    • Future: The collection is currently closed and will open in the future.
    • Open: The collection is open, and there are no extensions.
    • Extended: The collection is open and there is an extension for one or more organisations.
    • Closed: The collection is closed, and returns cannot be submitted.
  • Deadline: The last date any organisation can submit a return. This includes extensions.
  • Channel: the type of collection, such as statistics, grants, fund, or other (to be confirmed).
  • Fields: the number of fields in the form, used as an indication of collection complexity.
  • Form, Excel and Bulk: The number of returns broken down by submission method.
  • Organisations: the number of unique organisations who have a return (with any status).
  • Records: the total number of records across all submissions, including the average number of records per submission.
  • Errors: the percentage of organisations who when submitting a return and experienced one or more validation errors.

Discuss

  • Better name for type or channel: The grouping of collections should be based on their division, for example, Funding Service.
  • No start date: There is not enough space to display the start date.
  • Links in table data: Add links to the collection entity, submission report, and record store.
  • Collection group in table: This would allow users to filter and compare historic collections, particularly error rates.
  • Detailed error report: A collection activity log is needed to track errors in more detail.
  • Error rates: Errors should be categorised into types, such as "system" and "validation".
  • Access groups: A new user privilege is needed: "Can see all collections and associated data but cannot edit or take action."
