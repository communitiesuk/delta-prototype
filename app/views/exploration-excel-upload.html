Delta bulk upload errors

Updated: Thursday 13 February 2025

Proposal structure

  • What do we mean by bulk uploads?
  • What is the problem? What are the errors?
  • What is the scale of the problem? Are things improving or getting worse?
  • High-level heuristic review of the existing Orbeon bulk upload process (template download, data entry, form data upload, error reporting, and help and guidance).
  • A proposal and concept for a global collections MI report.
  • A proposal and concept for a collection manager activity report with error reporting.
  • Given Orbeon is central to bulk uploads, what can we realistically change in the user experience?
  • What does success look like (north star)?
  • 80:20 rule – how can we deliver 80% of the value in 20% of the time? What can we ship soon – in weeks, not months?
  • Thinking bigger – how can this project be approached to support wider, greater adoption of Delta?
  • A case for renaming this work from "bulk upload" to "Excel upload" to make Delta more versatile.

Collections MI report [concept]

Knowledge is power!

Access to clear and actionable insight is essential to understanding and resolving issues with bulk uploads. This inight would also facilitate wider improvements across Delta.

We believe the creation of a Management information (MI) reporting system be included as part of this project.

! Warning This is not a replacement for the Submissions Dashboard.

What problems does this solve?

  • Accessible to all*: For the first time ever one-click access to a global list of all collections, which can be reordered, with real-time insights.
  • Public-facing collection page: One-click access to view the public-facing collection entity page.
  • Form complexity: A high-level overview of the form's complexity, based on the amount of data requested.
  • Collection channel and data upload options: Ability to view and filter by collection channel (for example, Funding Service) and supported data upload options.
  • Submission data: High-level submission data with links to the submission report.
  • Record store view: One-click access to a filtered record store view for the collection.
  • Error rate: Provides insights to support process improvements and enable timely, targeted support from the relevant collection manager.

Collection report concept

Help with table labels
  • Channel: the type of collection, such as statistics, grants, fund, or other (to be confirmed).
  • Fields: the number of fields in the form, used as an indication of collection complexity.
  • Form, Bulk and API: The number of organisations that have submitted a return (not individual records) broken down by submission method.
  • Returns: the number of submissions from unique organisations.
  • Records: the total number of records across all submissions, including the average number of records per submission.
  • Errors: the percentage of organisations that submitted a return and experienced one or more errors.

Discuss

  • Better name for type or channel: The grouping of collections should be based on their division, for example, Funding Service.
  • No start date: There is not enough space to display the start date.
  • Links in table data: Add links to the collection entity, submission report, and record store.
  • Collection group in table: This would allow users to filter and compare historic collections, particularly error rates.
  • Detailed error report: A collection activity log is needed to track errors in more detail.
  • Error rates: Errors should be categorised into types, such as "system" and "validation".
  • Access groups: A new user privilege is needed: "Can see all collections and associated data but cannot edit or take action."

What good could look like

From the perspective of data provider…

  • Download Excel template: Users can download a pre-formatted Excel template.
  • Enter data into the Excel template: The template includes guidance and inline validation.
  • Upload Excel file: Users can upload the Excel file without needing to convert it to CSV.
  • Successful upload: A confirmation page is displayed.
  • Errors in submission: An error report is generated.
    • The error report should highlight which cells contain errors (for example, B36) and provide guidance on how to fix them (such as incorrect data type, out of range, or missing information).
    • Users should see an inline preview of the error data without needing to download or open the file.
  • Fixing errors: Users can either correct the errors in Excel and reupload the file or edit the data in the Orbeon form view.
